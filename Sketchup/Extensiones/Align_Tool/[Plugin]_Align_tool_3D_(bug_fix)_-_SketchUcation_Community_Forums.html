i also wondered about that but later realized i was not 'working properly'. in many cases all you need is to set the ending system; in this cases go for align group/component (as it takes the axis alignment of the group/component as the origin system); it saves you half of the work involved in the process. in cases when you do need to set the origin, then choose align and set both systems. does it make sense to you?

by Didier Bur » Sat Sep 04, 2010 10:05 am

Hi,

Doc says: "...you'll find a menu item under the "Tools" menu",..."if you select some entities and right-click, you'll find an "Align" option in the context menu, "if you select a group or a component and right-click, you'll find an "Align group/component" option in the context menu".

So if you don't have any of these opttions in your menus, there is something wrong.

Please open the Ruby Console and type: $align_loaded (Enter)
Please report what it says then.
